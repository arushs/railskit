# Configuration Reference

RailsKit uses `railskit.yml` at the project root as the single source of truth. The interactive setup wizard (`bin/setup`) generates this file, or you can edit it by hand.

---

## railskit.yml

Here's the full structure with all options:

```yaml
app:
  name: "RailsKit"           # Your app name
  domain: "localhost"         # Primary domain

database:
  adapter: "postgresql"       # postgresql | supabase | convex

auth:
  provider: "devise"          # devise | supabase | clerk
  google_oauth: true          # Enable Google OAuth login
  magic_links: true           # Enable magic link login

payments:
  provider: "stripe"          # stripe | lemon_squeezy

email:
  provider: "resend"          # resend | postmark | smtp
  from: "RailsKit <noreply@yourdomain.com>"

ai:
  provider: "openai"          # openai | anthropic | google | ollama
  model: "gpt-4o"             # Default model for agents

theme:
  primary_color: "#6366f1"    # Hex color for UI accent
  dark_mode: true             # Enable dark mode
```

## How Config Is Loaded

### Rails (API)

`api/config/initializers/railskit.rb` loads `railskit.yml`, deep-merges with defaults, and exposes `RailsKit.config`:

```ruby
RailsKit.config.app.name         # => "RailsKit"
RailsKit.config.ai.provider      # => "openai"
RailsKit.config.ai.model         # => "gpt-4o"
RailsKit.config.database.adapter # => "postgresql"
RailsKit.config.auth.provider    # => "devise"
RailsKit.config.auth.google_oauth # => true
RailsKit.config.payments.provider # => "stripe"
RailsKit.config.email.provider   # => "resend"
RailsKit.config.theme.primary_color # => "#6366f1"
```

Missing keys fall back to defaults:

```ruby
# Defaults (from RailsKit::DEFAULTS)
{
  app: { name: "RailsKit", domain: "localhost" },
  database: { adapter: "postgresql" },
  auth: { provider: "devise", google_oauth: true, magic_links: true },
  payments: { provider: "stripe" },
  email: { provider: "resend" },
  ai: { provider: "openai", model: "gpt-4o" },
  theme: { primary_color: "#6366f1", dark_mode: true }
}
```

### React (Frontend)

`web/src/config.ts` loads config from three sources (highest priority first):

1. **`VITE_RAILSKIT_*` env vars** — e.g., `VITE_RAILSKIT_APP_NAME`, `VITE_RAILSKIT_AI_PROVIDER`
2. **`web/src/railskit.generated.json`** — generated by `rails railskit:generate_frontend_config`
3. **Hard-coded defaults** (mirrors `railskit.yml` defaults)

Generate the JSON file:
```bash
cd api && bin/rails railskit:generate_frontend_config
# → writes web/src/railskit.generated.json
```

Use in React:
```typescript
import config from "@/config";

config.app.name       // "RailsKit"
config.ai.provider    // "openai"
config.theme.dark_mode // true
```

## Environment Variables

The `.env` file holds secrets and provider credentials. `bin/setup` generates it automatically.

### Core

| Variable | Description | Example |
|---|---|---|
| `RAILS_ENV` | Rails environment | `development` |
| `DATABASE_URL` | PostgreSQL connection string | `postgres://user:pass@localhost:5432/railskit_dev` |
| `REDIS_URL` | Redis (optional, for caching) | `redis://localhost:6379/0` |
| `SECRET_KEY_BASE` | Rails secret key | Generate with `rails secret` |
| `CORS_ORIGINS` | Allowed frontend origin | `http://localhost:5173` |
| `FRONTEND_URL` | Frontend URL (for redirects) | `http://localhost:5173` |

### Auth

| Variable | Description |
|---|---|
| `DEVISE_JWT_SECRET_KEY` | JWT signing secret (falls back to `SECRET_KEY_BASE`) |
| `GOOGLE_CLIENT_ID` | Google OAuth client ID |
| `GOOGLE_CLIENT_SECRET` | Google OAuth client secret |

### Payments

| Variable | Description |
|---|---|
| `STRIPE_SECRET_KEY` | Stripe secret key |
| `STRIPE_WEBHOOK_SECRET` | Stripe webhook signing secret |

### AI Providers

| Variable | Provider |
|---|---|
| `OPENAI_API_KEY` | OpenAI |
| `ANTHROPIC_API_KEY` | Anthropic |
| `GEMINI_API_KEY` | Google Gemini |
| `OLLAMA_BASE_URL` | Ollama (default: `http://localhost:11434`) |

### Email

| Variable | Provider |
|---|---|
| `RESEND_API_KEY` | Resend |
| `POSTMARK_API_TOKEN` | Postmark |
| `SMTP_ADDRESS`, `SMTP_PORT`, `SMTP_USERNAME`, `SMTP_PASSWORD` | SMTP |

### Database (Non-PostgreSQL)

| Variable | Provider |
|---|---|
| `SUPABASE_URL` | Supabase |
| `SUPABASE_ANON_KEY` | Supabase |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase (server-side) |
| `CONVEX_DEPLOYMENT` | Convex |

### Frontend

| Variable | Description |
|---|---|
| `VITE_API_URL` | API base URL for the React app |
| `VITE_WS_URL` | WebSocket URL (defaults to `VITE_API_URL` with `ws://` scheme) |

## Changing Providers After Setup

Edit `railskit.yml` and restart:

```yaml
# Switch from OpenAI to Anthropic
ai:
  provider: "anthropic"
  model: "claude-sonnet-4-20250514"
```

Then update `.env`:
```bash
ANTHROPIC_API_KEY=sk-ant-...
```

And restart: `bin/dev`

The `RubyLLM` initializer reads `RailsKit.config.ai.provider` at boot and configures the right API key. No code changes needed.

---

## Config Reload

In development, call `RailsKit.reload!` to re-read `railskit.yml` without restarting:

```ruby
RailsKit.reload!
RailsKit.config.ai.model  # picks up the new value
```

Provider modules also support reload:
```ruby
EmailProvider.reload!
DatabaseAdapter.reset!
```
